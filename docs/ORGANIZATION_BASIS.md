# Machine Organization, Consolidation & Backup Basis

Dataset-driven basis for organizing, deduplicating, and backing up the
machine. The authoritative dataset is `reports/machine_inventory.json`
(generated by `tools/machine_inventory.py`; digest in
`reports/machine_inventory.md`). Everything below is keyed to that dataset so
each action is reviewable before execution.

## Scope honesty

This scan ran in the **remote session container**. It fully covers the
repository and the container toolchain; it does **not** see the operator's
Linux Mint machine (`/home/jmmint`) or `/root/embroidery_business_agent_system`.
To produce the same dataset there, run on that machine:

```bash
EMBIZ_SCAN_ROOTS=/home/jmmint:/root python3 tools/machine_inventory.py
```

## Backup procedure (implemented, executed)

`scripts/backup_machine.py` produces per-run under `/home/user/backups/<ts>/`:
`repo.bundle` (full git history, bundle-verified), `worktree.tar.gz`
(complete working tree incl. gitignored runtime evidence), `MANIFEST.json`
(sha256 for every file and both archives). Executed 2026-07-16:
105.6 MB bundle + 97.8 MB tarball, both checksum-verified. The pushed GitHub
remote is the continuous offsite copy of tracked content.

**AWS stage: ON HOLD at operator request** — the operator will supply AWS
credentials. This environment's `AWS_*` variables are egress-proxy plumbing
(STS rejects them); no upload was performed or faked. When credentials
arrive: `python3 scripts/backup_machine.py --upload s3://<bucket>/embiz`
(STS-verifies identity before any write; uploads bundle, tarball, manifest
with sha256 object metadata).

## Standing rules

1. Original customer artwork is never committed (`input_images/` ignored).
2. Generated artifacts (contracts, dashboards) are rebuilt, never hand-edited.
3. Runtime state (`local_agents/state/`) is durable on the host, never
   committed.
4. Every dedupe/reorganization action must cite its dataset row
   (`machine_inventory.json` duplicate group) before deletion.
5. Secrets live only in environment files (mode 600); the inventory records
   their locations, never their contents.
